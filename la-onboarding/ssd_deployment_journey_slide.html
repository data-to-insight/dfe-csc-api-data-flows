SSD deployment journey
High level view of stages and who is involved at each step
Stage 1
Setup and smoke tests
  • Deploy SSD core tables and API staging tables
  • Run local smoke tests with fake data and schedule 1 to 1 support
Stage 2
Local verification
  • Visual checks that SSD tables are populating as expected
  • Cross check SSD staging cohort against recent 903 and CiN
Stage 3
First live data
  • Send an initial live alpha from SSD staging with filters if needed
  • Provide access to the private dashboard once live data is available
Stage 4
Operational rollout
  • Move from local click to run scripts to server deployment
  • Set up overnight or scheduled runs for ongoing operation
